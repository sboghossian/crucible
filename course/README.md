# AI-Assisted Development in 2026: A Practical Course

*10 modules. Built from the Crucible research study.*

---

This course was produced as a direct output of the Crucible research system. Every data point, every claim, every framework was run through the Debate Council before appearing here. The course is both a product of the research and a document of how adversarial AI review changes the outputs.

## Who This Is For

- Software engineers currently using AI coding tools who want to understand what they're actually using
- Engineering managers deciding how to structure AI-augmented teams
- Developers interested in multi-agent systems who want to go beyond demos
- Anyone building with AI who wants to make better decisions about AI

## What You'll Build

By Module 4, you'll have a running multi-agent system using Crucible. By Module 8, you'll have a prototype of an agent society with persistent identities and an XP economy. By Module 10, you'll have a framework for thinking about your own role as the landscape continues to shift.

## Prerequisites

- Python 3.11+
- Anthropic API key
- Familiarity with Python (intermediate level)
- Curiosity about where this is all going

## How to Use This Course

**If you have 2 hours:** Read modules 1, 5, and 10. These are the three that will most change how you think.

**If you have 8 hours:** Do the full course. The hands-on modules (4 and 8) will take the longest.

**If you're a manager:** Focus on modules 1, 6, 7, and 10. The technical modules are accessible but optimized for practitioners.

**If you're building a multi-agent system:** Modules 3, 4, 5, and 8 are your core path.
